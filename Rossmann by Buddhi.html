Fastai Implementation of Kaggle Competition: Rossmann Store Sales 3rd Place Winner¶

In this post, I shall talk about the implementation of the Rossmann Kaggle competition 3rd place winning solution. It was implemented by Cheng Guo as part of team Neokami Inc. Unlike most other solution which employed XGBoost, he implemented a neural network taking advantage of a method called Entity Embedding. The proposed solution compromised of minimal feature engineering.

We learnt his solution as part the Fastai Deep learning 1 course where we were learnt how to solve structured data problems using the fastai library. The fastai libraries wraps around pandas, numpy, pytorch and other libraries, but these three being the main ones for structurd neural networks.

Data provided by the Kaggle competition were:

1. train.csv - Historical data including sales
2. test.csv - Historical data excluding sales
3. store.csv - Supplemental information about the stores

The competition allowed participants to use external data in the condition that the data used was share with other. Guo used the following extra dataset which were provided by other participants.

1. store_states.csv - Maps stores to the state its in
2. state_name.csv - Maps state abbreviations to names
3. googletrend.csv - Google trends by Bundesland for each state
4. weather.csv - Weather conditions for each states

1. fastai.structured - Fastai library implementation along side with pandas for structured data implementations. This library is taught in the Machine Learning course.

Lets, starts joining tables now. Pandas does this using a merge function, we will be applying a left join. We started off by creating a simple function to do this, it simply appends "_y" to duplicated columns on the right table.

In [11]:
def join_df(left, right, left_on, right_on=None, suffix='_y'):
    if right_on is None: right_on = left_on
    return left.merge(right, how='left', left_on=left_on,
                      right_on=right_on, suffixes=("",suffix))

Durations¶

Next, we add a few new columns: These columns specify how long it has been since/to either a holiday or promotion. These specific columns we selected to specify since/to are called events. In time series data, it always helps to show how longs its been to and from an event as the neural network can then pick certain parterns from them, these patterns could be Eg: there might be a spick in sales just before a public holiday, dip in sales before a promotion, how long it takes for sales to rise after a promotion has started etc.

You don't need to append these rows as the neural network can learn them, but providing this data does make the learning process easier.

Firstly, we will sort the data appropriatly, iterate through every row and append a new column specifing the number of days to or from the event. Jeremy Howard (fastai founder and lecturer) wrote a very simple function to do this called get_elapsed() which takes a field name and the prefix that will be used to identify the new column.

I'd also like to point out that its quite facinating how easy python makes these kinds of data manuplilation possible. I remember writing similar code in SQL being quite difficult.

def get_elapsed(fld, pre):
    day1 = np.timedelta64(1,'D')
    last_date=np.datetime64()
    last_store=0
    res = []
    
    for s,v,d in zip(df.Store.values, df[fld].values, df.Date.values):
        if s!=last_store:
            last_date = np.datetime64()
            last_store=s
        if v: last_date=d
        res.append(((d-last_date).astype('timedelta64[D]')/day1))
    df[pre+fld]=res

Rolling Function¶

Basic idea here is were're going to generate a rolling average figure for the holidays and promotions within a window of 7 days. This gives an idea of the events status with a 7 day period and can signify weather an event is about to start, started, ended and how long its been since it ended within that window.

Again, this is useful information for the neural network that would allow it to pick certain patterns following a certain event.

For the following feilds:

1. Promo
2. School Holiday
3. State Holiday

We create two addition columns for each feature. 'event_bw' gives a rolling sum from the left edge of the window when the dates are sorted in ascending order, while 'event_fw' gives a rolling sum from the left edge of the windows when the dates are sorted in decending order. 'event_fw' gives us an idea of how much longer an event would last for within a weeks period while 'event_bw' gives as idea how long it has been since an event started within a weeks period.

Next we apply a fastai function called proc_df, this is pretty much the last step for data preperation. The function normalizes the data (do_scale=True), deal with missing values by filling them with their median and appending a new boolean column signifying which row has been updated with the median, then splits the dataset into dependent and independent which are the first two variables in the output.

The next variable output is the dictionary whose keys are the names of the columns with missing values and their medians. The last output is the object which contains the means and standard diviations used for normalization the training set. The last two output variable will be used as inputs when applying proc_df on the test set, this will ensure that the training set and the test set will be normalized using the same mean and standard diviation. Also, if it has missing values in the same columns, these would be updated with the median calculated in the training set.

If the test set has different columns with missing values, it will updated the existing dictionary, then the function will have to be applied to both the datasets again, but this time both the dictionary and the normalization mapper object will have to be passed as inputs into the function generating the training set.

Neural Network¶

What is a neural network: As Jeremy Howard simply explains it in the lectures, it consists of simple linear layers interspersed with a number of simple non-linear layer. When these kinds of layers are combined, they form what is called an universal approximation therom. According to the therom, these kinds of function can solve any given problem to arbitrarily close accuracy as long as theres enough parameters in the function.

So, we'll now implement a quick neural network to predict the sales of the Rossmann Stores in Germany given the data we just processed.

Remembering the rules of the competition, the metric used is Root-Mean_Squared_Percentage_Error. Here we define the function that calculates the RMSPE given an array of predicts and the array of actual values.

def inv_y(a): return np.exp(a)

def exp_rmspe(y_pred, targ):
    targ=inv_y(targ)
    pct_var=(targ-inv_y(y_pred))/targ
    return math.sqrt((pct_var**2).mean())

The fastai library provides us with a function which aids with finding the optimal learning rate. The fastai community goes through the latest papers on machine learning and everytime they find an interesting paper on how to improve a specific area. Its added into their library, this is a major advantage of using this library, its up to date with the latest alogrithms available

This specific technique was extracted from a 2015 paper on Cyclical Learning Rates for Training Neural Networks. The algorithm simply increases the learning rate by small increments every mini batch until the loss function stops decreasing. It's recommended to choose a learning rate just before the validation loss reaches its minimum, we'll see the reason to this in a few line.

The goal of alogrithm is to vary the learning every cycle (set to 1 epoch) by reducing the learning rate (learning rate annealing) which is a very common learning technique, here we use one half of the cosine function. This will force the model to find a stable and accurate area in the weight space. By resetting the learning rate, the model will jump to different space if its in a "spiky" area in the weight space.
